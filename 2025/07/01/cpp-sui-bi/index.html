写在前面

目前C++发展迅速，现代C++已与过去有极大的不同，在不断的发展中我们也需要不断的学习，此篇将记录一些我认为可能平时可能用到或需要用到的一些C++语法糖。

常量表达式（constant expression）

指在编译时就能计算出结果的表达式。C++11引入了constexpr关键字来显式定义常量表达式。

consexpr变量

constexpr int MAX_SIZE = 100;         // 基本类型
constexpr double PI = 3.1415926;      // 浮点数
constexpr const char* NAME = "C++";   // 字符串字面量

constexpr函数

C++11初次引入常量表达式，对constexpr函数限制比较严格，仅允许包含单个return语句，仅能返回字面类型（Literal Type）（如 int、float、constexpr 对象等），不能返回void,不能有分支、循环，允许使用三元运算符?:和递归。

constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}
// 编译时计算
constexpr int fact_5 = factorial(5);  // 120

C++14中允许使用多行语句，可以使用控制语句与定义局部变量(但不能是静态变量)，允许返回非 const 对象（只要它本身是 constexpr 构造的），允许 void 返回类型（但必须返回 void），开始支持std::max,std::min等库函数。

constexpr int count_zeros(int n) {
    int count = 0; // 允许局部变量
    while (n > 0) { // 允许循环
        count += (n % 10 == 0);
        n /= 10;
    }
    return count;
}

C++17新增**if constexpr（编译期条件分支）**，允许在模板和 constexpr 函数中使用编译期条件判断，并新增Lambda 表达式可以是 constexpr。

template <typename T>
constexpr auto get_value(T t) {
    if constexpr (std::is_pointer_v<T>)
        return *t; // 编译期分支
    else
        return t;
}
//lambda表达式
constexpr auto square = [](int x) { return x * x; };
constexpr int x = square(5); // 25

C++20新增加关键字**consteval**强制编译期执行，允许 new 和 delete（但必须完全在编译期管理），std::vector、std::string 等支持 constexpr（但生命周期必须限于编译期）。

consteval int strict_square(int x) { return x * x; }
constexpr int x = strict_square(5); // OK
// int y = strict_square(some_runtime_var); // 错误！

constexpr int dynamic_example() {
    int* p = new int(42);
    int val = *p;
    delete p;
    return val;
}

constexpr std::vector<int> get_data() {
    return {1, 2, 3, 4, 5};
}
constexpr auto data = get_data(); // C++20 合法

智能指针

C++11引入了智能指针，使得程序员不必再管理指针的释放问题，常用的有三种智能指针，包括std::unique_ptr、std::shared_ptr和std::weak_ptr，智能指针其行为像指针，可以使用如*ptr解引用。

std::share_ptr

std::share_ptr是一个类模板,它能记录有多少个对象共享管理指向的内存对象，多个std::share_ptr可以指向同一块内存，当最后一个std::share_ptr被销毁时内存随之释放。用户可使用make_share函数来创建，其有共享特性，自然支持拷贝构造及拷贝赋值。通常用于创建资源比较耗时用于共享资源。

//使用方法
std::share_ptr<int> int_ptr = make_share(new int(42));
std::share_ptr<int> int_ptr1(int_ptr);
int i = *int_ptr;
auto int_ptr_2 = int_ptr;
int_ptr.get();//当我们需要用到裸指针时可以使用get方法获取

std::share_ptr原理

share_ptr内部维护两个指针，一个指向引用计数，一个指向资源。

element_type*	   _M_ptr;         // Contained pointer.
__shared_count<_Lp>  _M_refcount;    // Reference counter.

基于此项原理我们就可以实现一个简单的share_ptr:

template<class T>
class SimpleSharePtr {
private:
    T *res_;
    int *ref_count_;
public:
    SimpleSharePtr();
    explicit SimpleSharePtr(T *data_ptr);
    SimpleSharePtr(const SimpleSharePtr<T> &old_ptr);
    SimpleSharePtr<T> &operator=(const SimpleSharePtr<T> &old_ptr);
    T& operator*();
    T* operator->();
    ~SimpleSharePtr();
    void release();
    T *get();

};
template<class T>
T *SimpleSharePtr<T>::operator->() {
    return res_;
}

template<class T>
T &SimpleSharePtr<T>::operator*() {
    return *res_;
}

template<class T>
void SimpleSharePtr<T>::release() {
    (*ref_count_)--;
    if (*ref_count_ == 0) {
        delete res_;
        delete ref_count_;
    }
}
template<class T>
SimpleSharePtr<T>::~SimpleSharePtr() {
    release();
}
template<class T>
T *SimpleSharePtr<T>::get() {
    return res_;
}
template<class T>
SimpleSharePtr<T> &SimpleSharePtr<T>::operator=(const SimpleSharePtr<T> &old_ptr) {
    if (this != old_ptr) {
        release();
        this->res_ = old_ptr.res_;
        this->ref_count_ = old_ptr.ref_count_;
        (*ref_count_)++;
    }
    return *this;
}
template<class T>
SimpleSharePtr<T>::SimpleSharePtr(const SimpleSharePtr<T> &old_ptr) {
    this->res_ = old_ptr.res_;
    (*(this->ref_count_))++;
}
template<class T>
SimpleSharePtr<T>::SimpleSharePtr(T *data_ptr) {
    res_ = data_ptr;
    ref_count_ = new int(0);
}
template<class T>
SimpleSharePtr<T>::SimpleSharePtr() {
    res_ = nullptr;
    ref_count_ = nullptr;
}

std::unique_ptr

顾名思义unique_ptr不支持多个对象指向同一块内存，自然也就不支持拷贝构造和拷贝赋值，但支持移动构造和移动赋值，比如

//当函数参数为std::unique_ptr<int> ptr
void test(std::unique_ptr<int> ptr){}；
//程序调用时如果为
test(ptr);//编译报错,实际会调用其拷贝构造
//可以使用std::move,比如
test(std::move(ptr));//编译通过，调用移动构造

可使用std::make_unique创建，当其本身被删除释放时，相应内存也被释放。使用方法与shared_ptr类似。

std::weak_ptr

weak_ptr是用来解决shared_ptr相互引用时的死锁问题,如果说两个shared_ptr相互引用,那么这两个指针的引用计数永远不可能下降为0,资源永远不会释放,比如下面定义两个类与测试函数：

class A;

class B
{
public:
    ~B()
    {
        std::cout << "B destory, a_ptr use_count:" << a_ptr.use_count() << "\n";
    }

       std::shared_ptr<A> a_ptr; //它会造成循环引用
    //std::weak_ptr<A> a_ptr;//它不会循环引用
};

class A
{
public:
    ~A()
    {
        std::cout << "A destory, b_ptr use_count:" << b_ptr.use_count() << "\n";
    }

     std::shared_ptr<B> b_ptr;//它会造成循环引用
    //std::weak_ptr<B> b_ptr;//它不会循环引用
};
A* TestB()
{
    std::shared_ptr<A> a(new A());
    std::shared_ptr<B> b(new B());
    a->b_ptr = b;
    b->a_ptr = a;
    return a.get();
}

类A有成员变量共享指针b,类B有成员变量共享指针a,可以分析TestB函数，当创建a时a引用计数为1，创建b时b引用计数为1,b赋值给a成员变量使得引用计数加1为2，同理a引用计数为2。那么当TestB执行完毕，对象a,b释放，导致引用计数减一，但此时两者引用计数均为1，也就不会调用相应析构函数导致内存泄漏。

weak_ptr可以解决这个问题是因为weak_ptr对对象的一种弱引用，不会增加对象的引用计数，因为这个特性其自然不会独立存在，必须依赖于一个shared_ptr，与shared_ptr之间可以相互转化，shared_ptr可以直接赋值给它，它可以通过调用lock函数来获得shared_ptr。
